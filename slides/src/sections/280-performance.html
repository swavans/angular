Plan for performance

or you will fail


  • Fix a list of devices (ex. iPhone 6+, Desktop IE11)
  • Fix network conditions
  • Fix acceptable numbers (ex. 3 sec to interaction)


It's a part of specifications!

You can't add performance later!

Startup Performance


  • Application Size
  • Compilation (AoT vs JiT)
  • Lazy Loading

Runtime Performance


  • Change Detection
  • Observables and Immutables

In most cases it is
not
an angular issue

It's like any other web application

General rules apply


  • Compress and minify content
  • Minimize round trips and data sent
  • Use simple CSS rules and minimize unused rules
  • Avoid querying dynamic DOM properties
  • Know how to profile
  • ...etc
