What is Webpack


Webpack is a highly configurable module bundler for javascript applications

Configuration via a JSON object
It relies on 4 concepts → Entry, Output, Loaders & Plugins

Entry

Webpack core concepts


Webapck creates a graph of all dependencies.

Entry → starting point of this graph

Output

Webpack core concepts


Once all your assets are bundled, webpack needs to know where to bundle it.

Output → how to treat bundled code

Loaders

Webpack core concepts


Webpack can handle plenty of different assets (css, html, images) but only understand Javascript.

Loaders → transform files into modules, then add to dependency graph.

  • Transpile Typescript to Javascript (ts-loader)
  • Transpile ES201x to ESx (babel-loader)
  • Inline images as data-url encoded string (image-loader)
  • CSS preprocessors (sass-loader, less-loader)

Plugins

Webpack core concepts


Loaders only work on a per-file basis.

Plugins → perform action on "chunk" of bundled modules.

Webpack & Angular


Everything is burried into Angular CLI

  • Development configuration (Typescript, Hot module replacement, Styles)
  • Production configuration (Typescript, Aot, UglifyJs, Compression)
